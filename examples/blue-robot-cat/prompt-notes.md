# Prompt Notes

Use these notes as a style and action reference. They are not a replacement for the final `hatch-pet` workflow.

For final production, convert the locked character into a `production_base`: a transparent or flat-background image that reads clearly inside one official `192x208` Codex pet cell. The production base should keep the face, ears, color blocks, badge, and compact silhouette, while simplifying tiny wires, text, glow effects, and shading.

## Character Lock

```text
Small half-body robot cat assistant, compact silhouette, screen-face eyes, tiny cat ears, electric blue and white shell, graphite gray joints, small code badge on chest, cute but slightly cyber, clean readable shape, no background, no text.
```

## Visual Style

```text
clean chibi mascot, soft rounded shapes, crisp outline, simple cel shading, friendly cyber details, expressive screen eyes, readable at small desktop-pet size
```

## Avoid

```text
realistic fur, horror face, heavy armor, complex cables, long written text, busy background, loose floating symbols, dramatic shadows
```

## Action Prompt Beats

### `idle`, 6 frames

```text
1 neutral pose, 2 screen eyes blink, 3 chest glow softly brightens, 4 ears twitch, 5 tiny breathing bob, 6 returns to neutral loop
```

### `failed`, 8 frames

```text
1 freezes, 2 eyes glitch, 3 ears droop, 4 body slumps, 5 tiny warning flicker, 6 weak restart blink, 7 screen stabilizes, 8 holds a tired failed pose
```

### `review`, 6 frames

```text
1 leans forward, 2 eyes scan, 3 code badge glows, 4 narrows eyes, 5 gives strict approving nod, 6 returns to focus
```
